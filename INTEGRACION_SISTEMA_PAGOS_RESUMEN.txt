NUEVAS CARACTERÍSTICAS IMPLEMENTADAS
═══════════════════════════════════════════════════════════════════════════════

✅ MODELO WALLET (Billetera del Conductor)
   - 1 conductor = 1 billetera
   - Balance: Saldo actual (puede ser negativo hasta límite)
   - Status: 'active' | 'blocked'
   - Límite negativo: -5000 COP (configurable)
   
   Métodos útiles:
   - puedeAceptarViajes(): Verifica si puede aceptar viajes
   - aplicarComision(): Descuenta comisión automáticamente
   - recargar(): Procesa recarga de saldo
   - actualizarBalance(): Recalcula balance desde transacciones

✅ MODELO TRANSACTION (Transacciones Financieras)
   - Fuente de verdad para todos los movimientos
   - Tipos: 'credit' (entrada) | 'debit' (salida)
   - Estados: 'pending' | 'completed' | 'failed'
   - Campos de auditoría: reference, description, timestamps
   
   Transacciones soportadas:
   - Comisiones por viaje
   - Recargas de saldo
   - Bonos (futuro)
   - Penalizaciones (futuro)

✅ SISTEMA DE COMISIONES POR PORCENTAJE
   - Comisión configurable (por defecto: 10%)
   - Se descuenta AUTOMÁTICAMENTE del balance del conductor
   - Se registra como transacción (debit) para auditoría
   - Integrado en PagoController.procesarViaje()
   
   FÓRMULA:
   comision = monto_total × (porcentaje / 100)
   balance_conductor = balance_anterior - comision

✅ BLOQUEO AUTOMÁTICO POR SALDO BAJO
   - Middleware EnsureWalletActive
   - Si balance < -5000 COP → status: 'blocked'
   - No puede aceptar viajes
   - Se desbloquea automáticamente al recargar
   - Respuesta 403: "Billetera bloqueada. Recarga saldo."

✅ ENDPOINTS DE WALLET (5 nuevos)
   GET    /api/wallet/balance          → Ver balance actual
   GET    /api/wallet/historial        → Historial de transacciones (paginado)
   GET    /api/wallet/resumen          → Estadísticas financieras
   POST   /api/wallet/recargar         → Iniciar recarga (prepara para Wompi)
   POST   /api/wallet/confirmar-recarga → Webhook (Wompi confirma)

✅ INTEGRACIÓN CON WOMPI (LISTO PARA PRODUCCIÓN)
   - Soporta: Nequi, Daviplata, PSE, Tarjeta Crédito/Débito
   - Flujo: Conductor → Solicita recarga → Wompi → Webhook → Balance actualizado
   - Transacciones 'pending' hasta confirmación
   - Bloqueo/desbloqueo automático según balance

FLUJO COMPLETO: VIAJE → PAGO → COMISIÓN → WALLET
═══════════════════════════════════════════════════════════════════════════════

1. PASAJERO SOLICITA VIAJE
   POST /api/viajes
   Estado: solicitado
   Precio: $30,000 COP

2. CONDUCTOR ACEPTA VIAJE
   PATCH /api/viajes/{id}/aceptar
   Estado: asignado
   Wallet conductor: $50,000 (sin cambios)

3. CONDUCTOR COMPLETA VIAJE
   PATCH /api/viajes/{id}/completar
   Estado: completado
   Precio total calculado: $30,000

4. PROCESAR PAGO
   POST /api/pagos/viajes/{id}/procesar
   Comisión: 10% = $3,000
   
   AUTOMÁTICAMENTE:
   ✅ Se crea Pago record
   ✅ Se crea Comision record
   ✅ Se crea Transaction (debit) en wallet
   ✅ Balance se recalcula: $50,000 - $3,000 = $47,000
   ✅ Status sigue: active

5. CONDUCTOR RECARGA SALDO
   POST /api/wallet/recargar
   Monto: $10,000
   Método: nequi
   
   AUTOMÁTICAMENTE:
   ✅ Se crea Transaction (credit, pending)
   ✅ Redirección a Wompi
   
6. WEBHOOK WOMPI CONFIRMA
   POST /api/wallet/confirmar-recarga
   reference: RECARGA-xxx
   status: completed
   
   AUTOMÁTICAMENTE:
   ✅ Transaction status: pending → completed
   ✅ Balance recalculado: $47,000 + $10,000 = $57,000
   ✅ Si balance >= -5000: status → active

NOTAS IMPORTANTES
═══════════════════════════════════════════════════════════════════════════════

⚠️  IMPORTANTE: Billetera permite balance negativo (hasta -5000 COP)
   Razón: Permite que conductor complete viaje aunque costo de comisión
           cause saldo negativo temporal
   Ejemplo: Balance $100, Viaje cuesta comisión $3,000
           → Balance resultante: -$2,900 (permitido)
           → Si balance < -5000 → blocked

✅ TRANSACCIONES: Todas las operaciones financieras crean registros auditable
   - Cada comisión = 1 Transaction (debit)
   - Cada recarga = 1 Transaction (credit)
   - Historial completo siempre disponible

✅ BALANCE RECALCULADO: Se suma desde transactions, no se actualiza en RAM
   - Fuente de verdad: tabla transactions
   - Balance en wallet es caché (para performance)
   - actualizarBalance() sincroniza

✅ EXTENSIBLE: Fácil agregar nuevos tipos de transacciones
   - Bonos: type='credit', description='Bono viajes completados'
   - Penalización: type='debit', description='Penalización servicio'
   - Reembolsos: type='credit', description='Reembolso viaje'
